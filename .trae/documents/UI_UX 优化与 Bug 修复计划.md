# 📅 修复日历视图 Bug 与 UI/UX 全面升级计划

## 1. 修复日历视图不一致 Bug
**问题**：界面上的"月视图"按钮默认呈激活状态，但日历组件初始化时使用的是"周视图"（Desktop）或"日视图"（Mobile），导致视觉与实际不符。
**解决方案**：
- 修改 `frontend/script.js` 中的初始化逻辑。
- 在创建日历实例前，根据屏幕宽度决定 `initialView`。
- **同步更新 UI 状态**：根据决定的 `initialView`，自动移除所有视图按钮的 `active` 类，并为当前实际使用的视图按钮添加 `active` 类。

## 2. Hello Kitty 风格 UI 全面精调 (🎀)
**目标**：打造精致、甜美且专业的界面，强化 Hello Kitty 元素，同时确保移动端体验完美。

### `frontend/index.html` 改动：
- **侧边栏 (Sidebar)**：
  - 在标题栏添加 **"新建会话"** 按钮 (✨)。
  - 优化装饰性 SVG（蝴蝶结），确保它们在不同屏幕下都能优雅展示。

### `frontend/style.css` 改动：
- **配色微调**：使用更柔和的樱花粉 (`#FFC0CB`) 和 蝴蝶结红 (`#ED0D92`) 进行点缀。
- **圆角与阴影**：增加卡片和按钮的圆角半径，使用更柔和的粉色系阴影。
- **移动端适配 (Exquisite Mobile)**：
  - 调整侧边栏宽度和遮罩层交互。
  - 优化日历工具栏在小屏幕上的布局。
  - 确保字体大小在手机上清晰易读。
- **装饰元素**：
  - 为"新建会话"按钮添加特殊的悬停动画。
  - 增强背景中蝴蝶结的漂浮动画效果。

## 3. 新增 AI 会话重置功能
**功能**：允许用户开启新的对话，并清除旧的上下文。
**实现**：
- 在 `frontend/script.js` 中实现 `resetSession()` 函数。
- **逻辑**：
  1. 弹出确认框（可选，或直接重置并提示）。
  2. 清除 `localStorage` 中的 `ai_session_id`。
  3. 清空聊天记录界面。
  4. 生成新的 Session ID。
  5. 自动发送一条新的欢迎消息（包含“当前会话已重置”的提示）。
- **交互**：点击侧边栏顶部的"新建会话"按钮触发。

## 4. 预期效果
- **Bug 修复**：进入页面时，视图按钮与实际日历视图严格对应（桌面端默认周视图，按钮高亮"周"；移动端默认日视图，按钮高亮"日"）。
- **视觉升级**：界面更加甜美温馨，Hello Kitty 元素自然融入，移动端操作流畅。
- **功能完善**：用户可以随时开启新话题，避免旧上下文干扰。
